# 🤖 Agent 架构设计笔记：ReAct, Claude Code 与 OpenClaw

本文档记录了关于现代 AI Agent（智能体）底层架构的探讨，特别对比了工业级重型框架 Claude Code 和轻量级开源框架 OpenClaw 的核心设计理念。这些内容可以作为后续沉淀项目技术文档、架构分享的参考素材。

---

## 1. 核心共识：ReAct Loop (推理-行动循环)

无论是 Claude Code、OpenClaw 还是由其衍生的 WeClaw，所有现代 Agent 框架的底层“发动机”都是一致的——**ReAct (Reasoning + Acting) 架构**。

**核心运行机制：**
1. **输入装配**：用户指令 + 系统上下文（Prompt）
2. **LLM 推理**：大模型独立思考当前状态，决定下一步动作
3. **工具调用 (tool_calls)**：执行具体的外部动作（如：读文件、查网络、写代码）
4. **观察结果 (Observation)**：将工具执行的输出喂回给大模型
5. **循环迭代**：大模型根据观察结果再次推理，决定是继续调用工具还是输出最终结论

> 💡 **架构隐喻**：ReAct 循环就如同汽车的“内燃机引擎”。Claude Code 相当于基于此引擎打造的“全自动驾驶特拉斯”，而 OpenClaw 则是提供标准底盘的“改装基座”。

## 2. Claude Code 的架构与精巧设计

Claude Code 代表了当前（资源雄厚、深度整合）闭源工业级 Agent 的顶级水平。它的核心竞争力远不止于 ReAct 循环，而是围绕循环构建的**上下文工程体系**和**深度工具链**。

### 精巧设计 1：KI 知识系统 (Knowledge Items) - 越用越聪明的“经验大脑”
*   **痛点**：每次开启新会话，AI 都会遗忘项目的特殊架构、代码规范或刚刚踩过的坑。
*   **设计**：系统内置一个后台运行的 Knowledge SubAgent。当你结束会话或取得关键进展时，SubAgent 会默默阅读对话记录，提取出有价值的代码规范、报错坑点或架构模式，**蒸馏（Distill）**成独立的 Markdown 知识项（KI）并落盘保存。
*   **使用**：新会话启动时，系统仅注入 KI 的“标题和摘要”。当 AI 发现某个问题与某个 KI 相关时，它会主动使用工具去 `read_file` 阅读完整的知识文档。它实现了跨会话的长期记忆和团队知识库级的沉淀。

### 精巧设计 2：微观到宏观的极度丰富的工具链 + 子 Agent 机制
*   **设计**：它不仅有基础的文件读写，还具备基于 AST（抽象语法树）的代码轮廓提取工具（`view_file_outline`）、精准替换工具（`replace_file_content`）。
*   **SubAgent**：当遇到需要操作网页的任务时，主 Agent 可以临时派生出一个专职的**浏览器子 Agent (Browser SubAgent)**，子 Agent 拥有控制浏览器的专用工具链，完成操作后向主 Agent 回报结果。

### 精巧设计 3：人机审批的安全分级 (Human-in-the-loop)
*   **设计**：并非所有工具都是危险的。Claude Code 在工具定义上引入了 `SafeToAutoRun` 属性。执行 `ls` 或读取文件时直接运行；执行 `rm` 打包或 `npm install` 这种能够改写系统状态的命令时，自动阻塞并向终端索要用户的确认（[Y/n]）。

## 3. OpenClaw 的架构与精巧设计

OpenClaw 作为一个开源框架，资源受限且需兼容多种外部平台，因此它的设计哲学是**“四两拨千斤”——用极简的底层工具组合出无限的扩展能力**。它的架构之美在于极致的“解耦”和“克制”。

### 精巧设计 1：技能的“渐进式披露” (Progressive Disclosure)
*   **痛点**：无限增加的插件会导致 System Prompt 瞬间爆炸，消耗天量 Token 且让 AI 失去注意力焦点。
*   **设计**：OpenClaw 采用标准化的 `SKILL.md`（包含 YAML 摘要 + Markdown 正文）格式。在组装初始 Prompt 时，**只注入几十个字符的 YAML 摘要**。当用户意图匹配到某摘要时，AI 按照约定**主动去读取该技能的完整文档**。
*   **价值**：实现了真正的“按需加载”，理论上可以让系统挂载无限个技能，极大地提升了可扩展性。

### 精巧设计 2：万物皆 Bash 的底层哲学
*   **痛点**：传统框架每接入一个新 API（如天气、股票、TickTick），都需要开发者编写 TypeScript/Python 插件代码，维护成本极高。
*   **设计**：OpenClaw 几乎不提供专用业务 API，而是赋予 LLM 最高级的原始工具：**`bash_execute`**。所谓的“增加新技能”，就是**写一篇 Markdown 教程教大模型怎么用 `curl` 调接口**。
*   **价值**：将“代码开发工作”降维打击成了“Prompt 编写工作”，人人都能给 AI 扩展系统能力。

### 精巧设计 3：指令拦截层与“幽灵命令” (Ghost Commands)
*   **痛点**：修改系统敏感配置（如修改 API Key），如果丢给大模型处理，不仅慢、耗费 Token，更有安全泄漏的风险。
*   **设计**：通过在 Agent Loop 外层增加一个前置拦截器，拦截 `/set`, `/get` 等指令。配置指令直接在本地内存/SQLite 闭环处理。
*   **价值**：大模型根本不知道这些指令发生过（幽灵特性），彻底隔绝了凭证进入上下文的隐患。

### 精巧设计 4：无 UI 的“通道”哲学
*   **设计**：抛弃了臃肿的前端面板，将自身定义为一个纯粹的“通道（Channel）后端”。无论接入 Terminal 还是接入微信公众号（如 WeClaw），都直接利用宿主平台的原生态消息流（支持 Markdown 渲染）作为交互界面。轻量化到了极致。

## 4. 总结

如果用武器库来做类比：

*   **Claude Code 是一座重型武库**：它依靠自研的顶级模型能力、厚重的工具堆栈（LSP 级代码分析、无头浏览器）以及跨会话的 KI 知识库系统，实现了极高的项目级自动化水平。
*   **OpenClaw 是一把瑞士军刀**：它不依赖特定的高阶 LLM，巧妙利用 `Bash + File I/O` 这一组原子能力，搭配 Markdown 教材级别的 Skill 体系，实现了以最小代码量博取最大的功能延展空间。

在后续 WeClaw 的演进中，**OpenClaw 的底层解耦**是我们需要坚持的基座，而 **Claude Code 的 KI 跨会话知识系统、人机安全拦截**则是我们可以逐步借鉴的优化方向。
